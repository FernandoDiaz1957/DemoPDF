1. Seleccionar modelos de referencia, puntos de vista y herramientas	

* Modelo de referencia: Trabajos anteriores o documentación relevante donde se haya
usado el TOGAF con el fin de poder implementar con desarrollo actual del trabajo.
* Puntos de vista: Los puntos de vista son perspectivas específicas desde las cuales se
examina la arquitectura y cada punto de vista se enfoca en ciertos aspectos o
preocupaciones de la arquitectura.
	- Diagrama de modelado conceptual
* Herramientas: Aplicaciones o técnicas que se utilizan para modelar y analizar la
Arquitectura de Datos.
	- ERDplus à Diseñar diagramas de entidad-relación online
	- Erwin à herramienta de modelado de datos
	- IDEF à Proporciona métodos de modelado para representar sistemas complejos

2. Desarrollar la descripción de la arquitectura de datos de referencia

* Descripción de la Arquitectura de Datos existente en la medida necesaria para
respaldar la Arquitectura de Datos de Destino
	- Modelo relacional
	- Modelo físico de datos

3. Desarrollar la descripción de la arquitectura de datos de Destino

* Mencionar las modificaciones de las actividades de los procesos o de las areas y mencionar lo planificado para dicha área, es decir, implementación de lo que se espera.
* Conceptualizar los bloques de construcción de arquitectura de datos.
* Basarse en las referencias escogidas anteriormente que incluyan el TOGAF, PETI (Plan Estratégico de Tecnologías de la Información), etc.

4. Realizar Análisis de Brechas

* Analizar las diferencias entre la arquitectura de referencia y la arquitectura objetivo
* Identificación y explicación las brechas entre la línea de base y el objetivo
* Especificación de que se mejora y como se hará, en caso haya brechas

5. Definir los Componentes Candidatos de la Hoja de Ruta

* Después de crear la Arquitectura de Referencia y la de Destino, se desarrolla una hoja de ruta para priorizar actividades futuras.
	OJO: Se puede realizar desde el principio indicando todas las actividades
* Esta hoja de ruta inicial se utilizará como base para una hoja de ruta consolidada en la fase de oportunidades y soluciones.

Se puede utilizar Diagrama de Gantt mediante Excel, Imágenes de la misma organización o MS Project -> Planificación de las tareas, su duración y programación a lo largo del tiempo

6. Resolver Impactos en el Paisaje Arquitectónico

* Comprender y abordar los impactos más amplios y oportunidades mediante la especificación de un contexto para una solución.
* Recomendable proponer cuadro de la intensidad de cada impacto para analizar la priorización de cada impacto.
* Recordar que esto se realiza en caso haya impactos.

7. Realizar una Revisión Formal de las Partes Interesadas

* Revisión de los dueños de la información y del arquitecto de datos
* Refinar la Arquitectura de Datos si es necesario
Ejemplo:

*Responsables involucrados en el procedimiento
En una tabla: en el indice: ENCARGADO y FUNCIÓN de estos 3 usuarios (USUARIOS NORMATIVOS, USUARIOS OPERATIVOS y ARQUITECTO de DATOS)

8. Finalizar la Arquitectura de Datos

* Al acabar todos los pasos se ha decidido aprobar la arquitectura de datos por medio del arquitecto de datos señalado anteriormente en la Fase Preliminar.
* No olvidar la firma.
